# Checklist de publicación

Antes de hacer público un notebook o marcarlo como listo, revisa este checklist.

## Contenido y autoría

- [ ] El notebook contiene únicamente mi propio código y explicaciones.
- [ ] Los enunciados privados del curso o del examen han sido eliminados o parafraseados en mis propias palabras.
- [ ] No se incluyen materiales del profesorado ni contenido protegido.
- [ ] Las conclusiones están escritas en mis propias palabras.

## Datos y privacidad

- [ ] No se publican datos personales ni sensibles.
- [ ] El dataset es público, sintético o su uso está claramente permitido.
- [ ] No hay rutas locales o privadas en el notebook (p. ej. `/content/drive/MyDrive/...` sin contexto).

## Salidas y tamaño

- [ ] Los outputs son útiles y comprensibles (no solo números sin explicación).
- [ ] Los outputs no son excesivamente grandes (evitar miles de líneas de log).
- [ ] Las imágenes incrustadas son razonables en tamaño.

## Nombre y estructura

- [ ] El nombre del archivo es profesional y descriptivo (en inglés, con guiones bajos).
- [ ] El README de la carpeta está actualizado.
- [ ] El badge "Open in Colab" está incluido o pendiente de activar.

## Calidad técnica

- [ ] El notebook explica el objetivo al principio.
- [ ] Las decisiones técnicas están justificadas (aunque sea brevemente).
- [ ] Las limitaciones están documentadas.
- [ ] El notebook se puede ejecutar de principio a fin sin errores en Colab (deseable).

## Ética

- [ ] Si el notebook toca datos de salud, incluye una nota aclarando que es educativo y no un diagnóstico.
- [ ] Los resultados no se presentan como definitivos ni como modelos de producción.
- [ ] Revisa [`ETHICS_AND_PRIVACY.md`](ETHICS_AND_PRIVACY.md) para más contexto.
